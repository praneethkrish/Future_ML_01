# 4️⃣ Feature Engineering
Feature engineering is the most important step in time-series forecasting.
## 📅 Date-Based Features
python
data["Day"]
data["Month"]
data["DayOfWeek"]
data["WeekOfYear"]
### Why?
Because sales depend on:
* Month (festive season)
* Day of week (weekend sales)
* Week number (seasonal patterns)
Machine learning models cannot understand dates directly.
So we convert them into numerical features.
## 🔁 Lag Features
`python
data["Lag_1"] = data["Sales"].shift(1)
data["Lag_7"] = data["Sales"].shift(7)
### What is Lag?
Lag means previous sales values.
Lag_1 → Yesterday’s sales
Lag_7 → Sales 7 days ago
### Why important?
Sales today depend on:
* Yesterday’s demand
* Same day last week
Without lag features, forecasting is weak.
## 📊 Rolling Mean Feature
python
data["Rolling_7"] = data["Sales"].rolling(7).mean()
Rolling_7 = Average of last 7 days.
### Why?
It smooths noise and captures short-term trend.
Very important for forecasting stability.
## Removing Null Values
python
data.dropna(inplace=True)
Lag and rolling create missing values at start.
We remove them.

# 7️⃣ Model Evaluation
python
mape = mean_absolute_percentage_error(y_test, predictions)
### What is MAPE?
MAPE = Mean Absolute Percentage Error
Formula:
[
MAPE = \frac{1}{n} \sum \left| \frac{Actual - Predicted}{Actual} \right| \times 100
]
### Interpretation:
| MAPE   | Meaning   |
| ------ | --------- |
| < 5%   | Excellent |
| 5–10%  | Very Good |
| 10–20% | Good      |
| >20%   | Poor      |

# 8️⃣ Recursive 30-Day Forecasting
This is the most important advanced part.
Instead of using fixed lag values, we:
1. Predict Day 1
2. Add predicted value to dataset
3. Use it as lag for Day 2
4. Repeat for 30 days
This is called:
🔁 Recursive Multi-step Forecasting
Why needed?
Because future actual values are unknown.
